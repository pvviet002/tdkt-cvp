Chuyên đề: Duyệt Bitmask

Duyệt tập con bằng thao tác bit (Bitmask Enumeration)

1 Tư tưởng cốt lõi (Core Ideas)

1.1. Sự tương đương giữa Tập con và Hệ nhị phân

Giả sử chúng ta có một tập hợp gốc gồm $N$ phần tử. Khi xây dựng một tập con, mỗi phần tử chỉ đứng trước hai sự lựa chọn: Được chọn vào tập con (1) hoặc Không được chọn (0).

Nếu ta gán mỗi phần tử tương ứng với một bit trong một chuỗi nhị phân độ dài $N$, ta sẽ thiết lập được một song ánh (1-1 mapping) hoàn hảo giữa mọi cấu hình tập con và các số nguyên từ $0$ đến $2^N - 1$. Chính vì vậy, ta có thể dùng một số nguyên (gọi là mask) để đại diện cho một tập hợp.

1.2. Các phép toán thao tác Bit (Bitwise Operations)

Thay vì dùng mảng boolean tốn kém bộ nhớ và thời gian, ta sử dụng trực tiếp các phép toán bit chạy trực tiếp trên thanh ghi của CPU (chỉ mất $O(1)$) để can thiệp vào các trạng thái cấu hình:

<< Phép dịch trái

Cú pháp 1 << N tương đương với phép tính $2^N$. Ta sử dụng phép toán này để sinh ra chặn trên cho vòng lặp chạy qua toàn bộ cấu hình.

& Kiểm tra trạng thái phần tử

Cú pháp mask & (1 << i). Biểu thức 1 << i tạo ra một mask phụ chỉ bật duy nhất bit $i$. Phép AND dùng để trích xuất bit đó. Nếu kết quả $\neq 0$, phần tử $i$ đang được chọn.

| Thêm phần tử vào tập (Bật bit)

Cú pháp mask | (1 << i). Phép OR đảm bảo bit thứ $i$ sẽ luôn trở thành $1$ bất chấp trạng thái trước đó của nó. Các phần tử khác giữ nguyên trạng thái.

^ và ~ Xóa / Đảo trạng thái

- Loại bỏ phần tử $i$: mask & ~(1 << i)
- Đảo ngược (Có thành Không, Không thành Có): mask ^ (1 << i)

1.3. Khuôn mẫu mã thuật toán (Code Pattern)

Một trong những ưu điểm mạnh nhất của Duyệt Bitmask là code cực kì ngắn gọn và không cần viết đệ quy quay lui (Backtracking). Cấu trúc kinh điển chỉ là 2 vòng lặp lồng nhau:

// Duyệt qua mọi tập con có thể có (từ 0 đến 2^n - 1)
for (int mask = 0; mask < (1 << n); ++mask) {
    
    // Khởi tạo các biến để lưu kết quả của tập con hiện tại
    // VD: int currentSum = 0;

    // Duyệt qua từng phần tử của mảng để kiểm tra xem nó có nằm trong 'mask' hay không
    for (int i = 0; i < n; ++i) {
        if (mask & (1 << i)) {
            // Nếu bit thứ i bật -> Phần tử A[i] thuộc tập con hiện tại
            // Thực hiện logic xử lý...
        }
    }
}

1.4. Đánh giá thuật toán & Ứng dụng

  • ▶ Độ phức tạp thời gian: Có tổng cộng $2^N$ trạng thái. Với mỗi trạng thái ta cần chạy vòng lặp $N$ bước để giải mã bit. Vậy độ phức tạp tổng quát là $O(2^N \cdot N)$.
  • ▶ Độ phức tạp không gian: Thuật toán sử dụng trực tiếp các số nguyên duyệt tuần tự nên chỉ tốn $O(1)$ bộ nhớ phụ, cực kì tối ưu.
  • ▶ Giới hạn áp dụng (Cực kì quan trọng): Vì là thuật toán hàm mũ (Exponential), kỹ thuật này chỉ khả thi trong thời gian quy định (thường là 1 giây) khi $N \le 20$. Tại $N=20$, số phép toán vào khoảng $20 \times 2^{20} \approx 2 \times 10^7$ (vẫn qua tốt). Nếu $N \ge 30$, thuật toán chắc chắn sẽ bị quá thời gian (TLE).

2 Bài toán kinh điển: Tổng tập con (Subset Sum)

Đề bài:

Cho mảng $A$ gồm $N$ số nguyên dương và số nguyên $S$. Hỏi có tồn tại một tập con nào của $A$ có tổng các phần tử đúng bằng $S$ hay không? In ra YES nếu có, ngược lại in ra NO.

Ví dụ mẫu:

Input: $N = 4, S = 5$
Mảng $A = [3, 1, 4, 2]$
Output: YES
Giải thích: Tập con gồm các phần tử {1, 4} có tổng bằng 5.

Mã nguồn C++:

#include <bits/stdc++.h>
using namespace std;

int n, S, a[25];

int main() {
    cin >> n >> S;
    for (int i = 0; i < n; ++i) cin >> a[i];

    bool found = false;
    for (int mask = 0; mask < (1 << n); ++mask) {
        int sum = 0;
        for (int i = 0; i < n; ++i)
            if (mask & (1 << i)) sum += a[i]; // Kiểm tra xem bit thứ i của mask có bật (bằng 1) không

        if (sum == S) found = true;
    }

    if (found) cout << "YES\n";
    else cout << "NO\n";

    return 0;
}
