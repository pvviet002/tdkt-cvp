Rolling Hashing & Thuật toán Rabin-Karp

Chuyên đề Lập trình thi đấu: So khớp chuỗi bằng kỹ thuật Băm (Hashing)

1 Tư tưởng cốt lõi

Mã băm (Hash) của một chuỗi là việc ánh xạ một chuỗi ký tự thành một số nguyên duy nhất (hoặc gần như duy nhất). Nhờ đó, thay vì so sánh hai chuỗi ký tự mất thời gian $O(K)$ (với $K$ là độ dài chuỗi), ta chỉ cần so sánh hai số nguyên đại diện cho chúng trong thời gian $O(1)$.

Công thức Polynomial Rolling Hash

Giả sử chuỗi $S$ có độ dài $L$, mã băm được tính bằng công thức đa thức:

$$H(S) = (S_1 \times B^{L-1} + S_2 \times B^{L-2} + \dots + S_L \times B^0) \pmod M$$
  • $S_i$: Mã ASCII của ký tự thứ $i$ (có thể dùng $S_i - \text{'a'} + 1$).
  • $B$ (Base - Cơ số): Thường chọn là một số nguyên tố lớn hơn số lượng ký tự phân biệt trong bảng mã (VD: $31, 311, 313$).
  • $M$ (Modulo): Thường chọn số nguyên tố lớn để đảm bảo giá trị băm nằm trong giới hạn kiểu số nguyên và giảm xác suất đụng độ (VD: $10^9+7, 10^9+9$).

Ví dụ minh hoạ: Xét chuỗi $S = \text{"code"}$ với hệ cơ số $B = 311$. Đặt giá trị các ký tự: $\text{'c'}=3, \text{'o'}=15, \text{'d'}=4, \text{'e'}=5$.

$$H(\text{"code"}) = (3 \times 311^3 + 15 \times 311^2 + 4 \times 311^1 + 5 \times 311^0) \pmod M$$

Tại sao gọi là "Rolling"? Kỹ thuật này cho phép tính mã băm của một đoạn con $[L, R]$ bất kỳ trong chuỗi $S$ với độ phức tạp $O(1)$ sau khi đã tiền xử lý mảng cộng dồn trong $O(N)$ bằng công thức:

$$Hash(L, R) = \big( Hash[R] - Hash[L-1] \times B^{R-L+1} \big) \pmod M$$

Minh họa trừ Hash (Rolling): Xét chuỗi $S = \text{"code"}$. Ta cần lấy mã Hash của đoạn $\text{"od"}$ ($L=2, R=3$).

Hash[3]
c $\times B^2$ + o $\times B^1$ + d $\times B^0$
- (Hash[1] $\times B^2$)
c $\times B^2$
= Hash(2, 3)
o $\times B^1$ + d $\times B^0$

* Lưu ý: Khi thao tác trừ Modulo, kết quả có thể bị âm. Cần cộng thêm $M$ trước khi lấy dư: (A - B % M + M) % M.

* Đụng độ (Collision): Là trường hợp hai chuỗi khác nhau nhưng lại có cùng mã băm. Để hạn chế tối đa rủi ro này, ta có thể áp dụng Double Hashing (tính mã băm độc lập với 2 cặp $(B, M)$ khác nhau).

2 Bài toán 1: Tìm kiếm chuỗi con (String Matching)

Đề bài: Đếm số lần xuất hiện

Cho chuỗi văn bản $S$ độ dài $N$ và chuỗi mẫu $T$ độ dài $M$. Hãy đếm số lần và in ra các vị trí mà $T$ xuất hiện dưới dạng chuỗi con liên tiếp của $S$.

  • $1 \le M \le N \le 10^5$.
  • Tất cả các ký tự là chữ cái in thường từ 'a' đến 'z'.
C++ (Rabin-Karp) 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 100005;
const long long BASE = 311;
const long long MOD = 1e9 + 7;

long long hashS[MAXN], powBase[MAXN];
string S, T;
int n, m;

// Hàm tính mã Hash của đoạn con [L, R]
long long getHash(int L, int R) {
    long long res = (hashS[R] - hashS[L - 1] * powBase[R - L + 1]) % MOD;
    return (res + MOD) % MOD; // Xử lý số âm
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    cin >> S >> T;
    n = S.length();
    m = T.length();

    // Để dùng chỉ số 1-based (thuận tiện cho prefix sum)
    S = " " + S;
    T = " " + T;

    // Tiền xử lý mảng lũy thừa
    powBase[0] = 1;
    for (int i = 1; i <= n; i++) powBase[i] = (powBase[i - 1] * BASE) % MOD;

    // Xây dựng mảng Hash tiền tố cho S
    hashS[0] = 0;
    for (int i = 1; i <= n; i++) hashS[i] = (hashS[i - 1] * BASE + S[i]) % MOD;

    // Tính mã Hash cho chuỗi T
    long long hashT = 0;
    for (int i = 1; i <= m; i++) hashT = (hashT * BASE + T[i]) % MOD;

    // Trượt cửa sổ độ dài m trên S
    for (int i = 1; i <= n - m + 1; i++)
        if (getHash(i, i + m - 1) == hashT)
            cout << i << " ";

    return 0;
}

3 Bài toán 2: Đếm đoạn xâu đối xứng (Palindromic Substrings)

Đề bài: Đếm đoạn xâu đối xứng

Cho trước một xâu $s$, hãy xác định xem có bao nhiêu xâu con của nó là xâu đối xứng (đọc từ trái sang phải giống như đọc từ phải sang trái).

  • Dòng duy nhất chứa xâu $s$ chỉ bao gồm chữ cái in thường.
  • Ràng buộc: $1 \le |s| \le 10^5$.
Ví dụ 1
Input: ata
Output: 4
Giải thích: a, t, a, ata
Ví dụ 2
Input: aabbaa
Output: 11

Phân tích thuật toán

Với giới hạn $N = 10^5$, thuật toán duyệt ngây thơ $O(N^3)$ hoặc mở rộng từ tâm $O(N^2)$ sẽ vượt quá thời gian cho phép (TLE). Ta có thể tối ưu độ phức tạp xuống $O(N \log N)$ bằng cách kết hợp Rolling Hash và Tìm kiếm nhị phân.

Hướng tiếp cận:

  1. Xây dựng mảng mã băm thuận cho $S$ và ngược cho xâu đảo ngược của $S$.
  2. Một đoạn $[L, R]$ là đối xứng khi và chỉ khi Hash Thuận = Hash Ngược. Thao tác kiểm tra này tốn $O(1)$ thời gian.
  3. Duyệt qua từng vị trí $i$ đóng vai trò là tâm đối xứng:
    • Tâm lẻ (ví dụ "aba"): Tâm là ký tự tại vị trí $i$.
    • Tâm chẵn (ví dụ "abba"): Tâm nằm giữa vị trí $i$ và $i+1$.
  4. Tính đơn điệu: Nếu một đoạn độ dài $2K+1$ đối xứng thì đoạn độ dài $2K-1$ có cùng tâm cũng đối xứng. Nhờ đó, ta có thể dùng Tìm kiếm nhị phân (Binary Search) để tìm độ dài (bán kính) đối xứng lớn nhất tại mỗi tâm thay vì duyệt tuần tự.

Trực quan: So sánh Hash Thuận và Hash Ngược đoạn [2, 4]

S:
a a b a c
Compare $HashF(2,4) == HashR(2,4)$
revS:
a a b a c
C++ (Hash + Binary Search) O(N log N) 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 100005;
const long long BASE = 311;
const long long MOD = 1e9 + 7;

long long hashF[MAXN], hashR[MAXN], powBase[MAXN];
string S;
int n;
long long ans = 0;

long long getHashF(int L, int R) {
    long long res = (hashF[R] - hashF[L - 1] * powBase[R - L + 1]) % MOD;
    return (res + MOD) % MOD;
}

// Chú ý cách ánh xạ chỉ số ngược
long long getHashR(int L, int R) {
    int L_rev = n - R + 1;
    int R_rev = n - L + 1;
    long long res = (hashR[R_rev] - hashR[L_rev - 1] * powBase[R_rev - L_rev + 1]) % MOD;
    return (res + MOD) % MOD;
}

bool isPalindrome(int L, int R) {
    return getHashF(L, R) == getHashR(L, R);
}

int main() {
    ios_base::sync_with_stdio(false); cin.tie(NULL);
    cin >> S;
    n = S.length();
    S = " " + S; // 1-based

    powBase[0] = 1;
    for (int i = 1; i <= n; i++) powBase[i] = (powBase[i - 1] * BASE) % MOD;

    // Hash thuận
    hashF[0] = 0;
    for (int i = 1; i <= n; i++) hashF[i] = (hashF[i - 1] * BASE + S[i]) % MOD;

    // Hash ngược
    hashR[0] = 0;
    for (int i = 1; i <= n; i++) hashR[i] = (hashR[i - 1] * BASE + S[n - i + 1]) % MOD;

    for (int i = 1; i <= n; i++) {
        // Đếm Palindrome tâm lẻ (ví dụ "aba")
        int l = 1, r = min(i, n - i + 1), max_rad = 0;
        while (l <= r) {
            int mid = (l + r) / 2;
            if (isPalindrome(i - mid + 1, i + mid - 1)) {
                max_rad = mid;
                l = mid + 1;
            } else r = mid - 1;
        }
        ans += max_rad;

        // Đếm Palindrome tâm chẵn (ví dụ "abba", tâm giữa i và i+1)
        l = 1; r = min(i, n - i); max_rad = 0;
        while (l <= r) {
            int mid = (l + r) / 2;
            if (isPalindrome(i - mid + 1, i + mid)) {
                max_rad = mid;
                l = mid + 1;
            } else r = mid - 1;
        }
        ans += max_rad;
    }
    
    cout << ans << "\n";
    return 0;
}
